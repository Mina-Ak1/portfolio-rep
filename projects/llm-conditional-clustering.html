Conditional Clustering using LLMs

Python GPT-4 LLM Embeddings K-Means PCA Data Viz

TL;DR. Same five texts, different clusters—depending on whether you group by topic, sentiment, or emotion. I compare a simple TF-IDF + K-Means baseline with task-specific LLM embeddings and a prompt-based approach. Result: the condition changes the geometry, and task-specific embeddings produce clearer, more intuitive clusters.

About the Data

Five short texts designed to be clusterable in multiple ways (sports/finance/science for topic; positive/neutral/negative for sentiment; joy/sadness/etc. for emotion). This makes it easy to see how the “definition of similarity” affects grouping.

What did I compare?

• Task-specific embeddings for each condition (topic, emotion, sentiment) + K-Means.
• Prompted LLM route (define the condition in natural language, then group).
• Classical baseline: TF-IDF + K-Means.

PCA views of the embeddings

Topic shows tighter, clearer groupings; sentiment and emotion are more diffuse (they’re more subjective), so clusters spread out more.

K-Means on LLM embeddings vs. TF-IDF

In the condition-aligned LLM spaces, K-Means generally recovers the expected groupings. TF-IDF, based purely on word frequency, can mix very different texts if they share similar vocabulary.

Do the spaces agree?

Only loosely. Topic space tends to differ from sentiment/emotion, while sentiment and emotion are more alike. Exact values aren’t the point—the takeaway is that the condition you care about changes what “near” means.

Takeaways

  • The condition defines the clustering—same inputs, different groups.
  • Task-specific LLM embeddings give cleaner clusters than TF-IDF.
  • Prompting works when you can state the condition directly.
